# file-time-magic — 文件时间属性修改技能 v2.0

> ⚠️ **平台要求**：本 skill 仅支持 **Windows 系统**。需要已安装 Python（用于运行脚本）和 PowerShell。

## 核心功能

| 能力 | 说明 |
|------|------|
| **编辑时长设置** | 自动处理编辑时长（带随机误差），并同步设置文件系统时间 |
| **用户指定时间点** | 支持用户指定创建时间、修改时间，系统自动计算编辑时长 |
| **时间逻辑自动计算** | 创建时间、修改时间、访问时间自动保持逻辑一致 |
| **随机性注入** | 分钟有±5误差，秒数随机，看起来更真实 |
| **时间上限保护** | 所有时间不超过当前时间 |
| **多种文件类型** | 支持 docx, pptx, xlsx（有编辑时长属性）和其他文件类型 |
| **文件夹支持** | 支持修改文件夹的创建时间、修改时间、访问时间（无内部编辑时长） |

## ⚠️ 核心原则（每次执行必读）

| 原则 | 说明 |
|------|------|
| **时间逻辑一致性** | 创建时间 + 编辑时长 ≤ 修改时间 ≤ 访问时间 |
| **随机性必须** | 用户说"2小时"，实际是115-125分钟；秒数也要随机 |
| **时间上限** | 所有时间不能超过当前系统时间 |
| **真实可信** | 工作时间在合理范围（早上8点-晚上10点） |

## 命令语法

```python
# 基本用法
python "{SKILL_DIR}/scripts/set_file_time.py" --file "<文件路径>" --edit-duration "<时长>"

# 指定创建时间
python "{SKILL_DIR}/scripts/set_file_time.py" --file "<文件路径>" --create-time "2024-01-15 09:30:00" --edit-duration "3小时"

# 指定修改时间
python "{SKILL_DIR}/scripts/set_file_time.py" --file "<文件路径>" --modify-time "2026-04-18 14:00:00" --edit-duration "90分钟"

# 指定创建和修改时间（自动计算编辑时长）
python "{SKILL_DIR}/scripts/set_file_time.py" --file "<文件路径>" --create-time "2024-01-15 09:00:00" --modify-time "2026-04-18 14:00:00"
```

**参数说明**

| 参数 | 必填 | 说明 |
|------|------|------|
| `--file` | 是 | 目标文件名路径 |
| `--edit-duration` | 否* | 编辑时长，如 "2小时"、"120分钟"、"3h" |
| `--create-time` | 否* | 创建时间，如 "2024-01-15 09:30:00" |
| `--modify-time` | 否* | 修改时间，如 "2026-04-18 14:00:00" |
| `--access-time` | 否 | 访问时间（可选） |
| `--base-time` | 否 | 参考时间（默认当前时间） |
| `--dry-run` | 否 | 只显示计算结果，不实际修改 |
| `--force` | 否 | 强制执行，跳过未来时间确认 |

\* 至少需要指定 `--edit-duration` 或 `--create-time`/`--modify-time` 其中之一

## 使用模式

| 用户输入 | 系统行为 |
|---------|---------|
| 只指定 `--edit-duration` | 从当前时间往前推算所有时间 |
| `--create-time` + `--edit-duration` | 从创建时间往后推算修改时间 |
| `--modify-time` + `--edit-duration` | 从修改时间往前推算创建时间 |
| `--create-time` + `--modify-time` | 自动计算编辑时长（时间差） |
| 三时间全指定 | 直接设置，忽略编辑时长计算 |

## 文件夹特殊情况

| 特性 | 文件 | 文件夹 |
|------|------|--------|
| 文件系统时间戳 | ✅ 创建/修改/访问 | ✅ 创建/修改/访问 |
| Office 内部编辑时长 | ✅ 有 (TotalTime) | ❌ 无 |
| 修改时间语义 | 最后编辑内容 | 最后添加/删除文件 |

**文件夹使用示例**

```python
# 修改文件夹的创建和修改时间
python "{SKILL_DIR}/scripts/set_file_time.py" --file "C:\工作材料" --create-time "2024-01-15 09:00" --modify-time "2024-06-20 18:00"

# 注意：文件夹不支持 --edit-duration 参数（无内部编辑时长）
# 但会自动计算文件系统层面的时间差
```

**注意事项**
- 文件夹的修改时间通常应该 ≥ 内部最新文件的修改时间
- 如果文件夹被占用（如资源管理器正在访问），修改可能失败
- 访问时间可能会立即被系统更新为当前时间

## 时间计算规则

1. **编辑时长随机化**
   - 用户说"X小时"：实际 = X×60 ± random(-5, +5) 分钟
   - 用户说"X分钟"：实际 = X ± random(-3, +3) 分钟
   - 秒数：random(0, 59)

2. **创建时间计算**
   - 工作时间范围：08:00 - 22:00
   - 如果创建时间不在这个范围，自动调整到最近的工作时间段

3. **修改时间计算**
   - 修改时间 = 创建时间 + 编辑时长 + random(0, 2) 分钟缓冲
   - 秒数随机

4. **访问时间计算**
   - 访问时间 = 修改时间 + random(3, 15) 分钟

## 时间格式支持

| 输入格式 | 示例 |
|---------|------|
| 标准格式 | "2024-01-15 09:30:00" |
| 无秒 | "2024-01-15 09:30" |
| 斜杠 | "2024/01/15 09:30" |
| 月-日 | "01-15 09:30"（默认今年） |
| 时间 | "09:30"（默认今天） |

| 时长格式 | 示例 |
|---------|------|
| 中文混合 | "2小时"、"2小时30分"、"两小时半" |
| 纯数字+单位 | "120分钟"、"3小时"、"1.5小时" |
| 简写 | "2h"、"3h30m"、"150m" |
| 纯数字 | "120"、"90"（默认分钟） |

## 示例

### 示例 1：只指定编辑时长

```bash
python set_file_time.py --file "报告.docx" --edit-duration "2小时"
```

**结果：**
- 编辑时长：122分59秒（有随机误差）
- 创建时间：今天 08:30:29（自动推算）
- 修改时间：今天 10:32:46
- 访问时间：今天 10:40:00

### 示例 2：指定创建时间 + 编辑时长

```bash
python set_file_time.py --file "报告.docx" --create-time "2024-03-15 09:00" --edit-duration "3小时"
```

**结果：**
- 创建时间：2024-03-15 09:00:xx（用户指定）
- 修改时间：2024-03-15 12:0x:xx（创建时间+3小时）
- 编辑时长：180分钟（Office内部）

### 示例 3：指定创建和修改时间

```bash
python set_file_time.py --file "报告.docx" --create-time "2024-01-15 09:00" --modify-time "2026-04-18 14:00"
```

**结果：**
- 创建时间：2024-01-15 09:00:xx
- 修改时间：2026-04-18 14:00:xx
- 编辑时长：自动计算（约228万分钟）

### 示例 4：dry-run 模式

```bash
python set_file_time.py --file "报告.docx" --edit-duration "1.5小时" --dry-run
```

## 处理边界情况

| 情况 | 处理方式 |
|------|---------|
| 文件不存在 | 返回错误，不执行 |
| 文件被占用 | 尝试关闭占用进程（Word/Excel/PPT），再执行 |
| 创建时间在修改时间之后 | 自动交换或调整 |
| 时间跨度超过文件存在时间 | 自动调整到合理范围 |
| 创建时间在工作时间外 | 自动调整到最近的工作时间（08:00-22:00）|
| 用户输入无效时间格式 | 提示正确格式 |
| **时间超过当前时间** | ⚠️ 发出警告，要求用户确认（除非 --force）|

## ⚠️ 未来时间警告

当用户指定的时间超过当前时间时，脚本会：

1. 显示警告信息
2. 列出所有"未来时间"
3. 要求用户确认是否继续

**示例输出：**
```
============================================================
⚠️  警告：以下时间超过当前时间
============================================================
  • 修改时间: 2026-04-18 14:00:45
  • 访问时间: 2026-04-18 14:00:45

当前时间：2026-04-18 12:06:00

这可能是一个不合理的设置（文件来自未来？）
------------------------------------------------------------
是否继续执行？(y/N): 
```

**跳过确认：**
```bash
# 使用 --force 参数跳过确认
python set_file_time.py --file "file.docx" --modify-time "2030-01-01 00:00" --force
```

## 验证修改结果

用户可以通过以下方式验证：

1. **文件系统时间**：右键文件 → 属性 → 详细信息
2. **Office 内部属性**：Word 打开 → 文件 → 信息 → 属性 → 高级属性 → 统计
3. **PowerShell 验证**：
   ```powershell
   Get-Item "文件路径" | Select-Object Name, CreationTime, LastWriteTime, LastAccessTime
   ```

## 注意事项

- 修改 Office 文件内部属性需要解压和重新打包，过程是安全的
- 如果文件被 Word/Excel/PPT 占用，会自动关闭这些进程
- 所有时间设置为本地时区（Asia/Shanghai）
- 临时文件会自动清理
- **v2.0 新增**：用户可指定创建时间、修改时间，系统自动计算编辑时长
